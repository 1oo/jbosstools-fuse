Exercising routes

Revision History

Overview

In addition to simplifying the process of creating and editing routes, Fuse provides tools to make it easier to run your routes and deploy them into test environments. Using Fuse you can:

  • run a route using Maven

  • run a route as a local Apache Camel context

  • deploy into a running OSGi container

Using Maven

You can run your route with Maven, using Maven's standard goals. This method closely simulates how an Apache Camel project would run outside of Fuse ESB Enterprise.

See Working with runtime profiles .

Running as a local Apache Camel context

Red Hat subscribers can run all routes in a context directly inside Fuse without setting up a Maven runtime profile. You can run routes locally with or without running validation tests. Running a context directly inside Fuse does not require that your project be a Maven project.

Deploying to OSGi

Red Hat subscribers can also deploy routes into a running OSGi container. Fuse supports deployment into these OSGi containers: Fuse ESB Enterprise v7.0, Apache ServiceMix v4.x, and Apache Karaf v2.x.

See Deploying Projects to a Container.
