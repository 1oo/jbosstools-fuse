Release Notes

Version 7.2

09 Sep 2012

New Features

These features are new in Fuse IDE v7.1:

Eclipse v 3.7

Fuse IDE v7.1 runs on Eclipse v 3.7 and works with m2e, the new Maven integration in Eclipse.

Full binary distribution

Using the Installer, you can now download the full binary distribution, which includes Eclipse v3.7.

Tip

You can still use the Eclipse plugin update mechanism to download Fuse IDE v7.2. If you do so, make sure you are running Eclipse v3.7; otherwise, Fuse IDE v7.2 will not run.

Fuse Fabric perspective

The Fuse Fabric perspective is the staging area for connecting to and managing fabrics. A fabric created by Fuse Fabric technology is a cluster of containers, each of which has an associated profile (and version) that determines what runs inside it. You can view, create, edit, and delete fabric details (connections to existing fabrics) and profiles; create and provision containers; and create versions. You can navigate fabrics to see their underlying JVMs. Open a terminal on any process in a fabric to have full Fuse ESB console access with colors and tab completion, without having to remember host/port names. Easily browse JMX, Fuse MQ, and Fuse ESB beans, endpoints, routes, and so on, without having to remember JMX URLs.

If using a JClouds-supported cloud provider (EC2, Rackspace, Slicehost, VMware, vSphere, ...), you can create, view, pause, resume, and reboot your compute nodes directly from Fuse IDE, and you can easily find and view all logs distributed across multiple fabrics.

Export to Diagram

You can now export your route diagrams as PNG, JPG, or BMP files to easily include them in presentations or documentation.

Enhancements

Fuse IDE v7.2 has an all new look and feel! The route editor has been overhauled, making it easier and more intuitive to design, create and edit routes, and the Fuse Integration perspective (formerly Fuse IDE) has been greatly enhanced, making it easier and more intuitive to trace messages and profile routes.

Route editor layout

The route editor displays only one route on the canvas at a time, so the Route box has been eliminated. Now you can freely move nodes around as you add more patterns to the canvas. Fuse IDE provides an easy mechanism, Outline view, for switching routes on the canvas. You can also switch between routes using the Routes menu on the menu bar.

Route editor connector tool

The route editor's connector tool is now part of each node on the canvas. Simply hover the mouse over a node to reveal its connector arrow, then drag and drop it on a target node to connect the two nodes.

Connectors now have bendpoints that break them into segments, enabling you to easily drag connectors around densely packed nodes in complex routes.

Route editor reuse of endpoints and beans

Beans and endpoints defined in the routing context file are added to the Endpoints group in the Palette, so they also appear in the Routes menu and the context menu's Add > Endpoints submenu.

Trackpad scrolling

Two-finger trackpad scrolling on the canvas has been improved.

Fuse IDE perspective

The Fuse IDE perspective has been renamed Fuse Integration to more clearly indicate its function.

Fuse Integration visualization

Visualizations of endpoints, routes, processors, queues, and topics have been combined with tables and charts, so you can easily see relative sizes.

Fuse Integration profiling

For route tracing, profiling and timing metrics have been added to both the Messages view and the Diagram view. Profiling identifies which steps in a route were invoked the most times and which steps took the most time to process messages. Timing metrics displayed in Diagram view include average time, count, max time, and min time.

Fuse Integration tabular views.

You can now easily adjust the order and width of the columns in all tabular views. To rearrange column order, you simply drag column headers to different locations in the table. To resize column width, you simply drag a column's boundary line to expand or contract the width. To hide a column from view, you simply contract its width completely.

Fuse Integration JVM statistics

Fuse IDE now provides JVM statistics views on JMX Explorer, so you can see MBeans, Timeline, Threads, Memory, and CPU statistics.

Fuse Integration deployment options

You can now easily deploy your projects to a Fuse ESB Enterprise 7.x server or to a fabric profile using the enhanced Deploy to... tool or drag & drop.

Known issues

The following are issues are known to exist in Fuse IDE 7.2:

Fuse IDE does not work in Spring STS

Fuse IDE cannot be installed into Spring STS eclipse distributions because Fuse IDE uses m2e, while Spring uses m2eclipse.

Debug As context menu option nonfunctional

Slated to become functional in v2.2.

You cannot view or modify a profile's configurations from within Fuse IDE

Slated to be fixed in v2.2.

m2e is incompatible with some Maven plugins

When you try to import examples, such as the cxf-codegen-plugin, using Fuse IDE's import wizard, you may encounter m2 connector errors and class not found errors. To workaround the incompatibility, for each affected plugin, add code like this for cxf-codegen-plugin to your Pom file: 

 <pluginManagement>
 
   <plugins>
     <!--This plugin's configuration is used to store Eclipse m2e settings only. It has no influence on the Maven build itself. -->
     
       <plugin>
          <groupId>org.eclipse.m2e</groupId>
          <artifactId>lifecycle-mpping</artifactId>
          <version>1.0.0</version>
          
          <configuration>
             <lifecycleMappingMetadata>
                <pluginExecutions>
                   <pluginExecution>
                   
                      <pluginExecutionFilter>
                      
                         <groupId>org.apache.cxf</groupId>
                         <artifactId>cxf-codegen-plugin</artifactId>
                         <versionRange>[2.3.3,)</versionRange>
                         
                         <goals>
                            <goal>wsdl2java</goal>
                         </goals>
                         
                      </pluginExecutionFilter>
                      
                      <action>
                         <execute />
                      </action>
                      
                   </pluginExecution>
                </pluginExecutions>
             </lifecycleMappingMetadata>
          </configuration>
          
       </plugin>
       
   </plugins>
   
 </pluginManagement>
