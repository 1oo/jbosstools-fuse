Deploying a Project to a Container's Deploy Folder

When Fuse deploys a project into a container's deploy folder, it builds the project as specified in the project's POM and copies the resulting package to a predetermined location. Fuse assumes two things:

  • the specified location is a container's deploy folder

  • the package generated by the project is compatible with the container

Before you can deploy a project to a container's deploy folder you must add the container's deployment information to Fuse. Once Fuse knows the container's information, the container will appear as an option in the Deploy To... menu.
