Name

Finally — Node traversed when a try, catch, finally block exits

Usage

The Finally pattern specifies a series of steps to take when a Try, Catch, Finally block has completed.

The Finally pattern can only occur following a Try pattern. It must be followed by either a processor or an endpoint.

Properties

Table 35 describes the properties you can specify using the properties editor.

Table 35. Finally Properties

NameDescription
IdSpecifies a unique identifier for the endpoint. The Id can be used to refer to the endpoint in the Camel XML file.
DescriptionSpecifies a text description for the node. This description is included in the generated XML file, but it is informational only. It is not used by Apache Camel.
