camel-mail

The camel-mail component provides interaction with email.

By default, Camel creates its own mail session and use this to interact with your mail server. However, EAP has already provided a mail subsystem with all the relevant support for secure connections, username and password encryption, etc. Red Hat recommends you to configure your mail sessions within the EAP configuration and use JNDI to wire them into your Camel endpoints.

For more information, refer the section Camel-Mail Configuration.
